# Case 006 — Mint Mobile Onboarding
*Filed: 2026-04-09 | Open Pit — cheap gun*

**What failed:** New phone plus new internet at the same time. Onboarding assumes working SMS on the phone you're replacing, working internet while setting up the internet, and a password that doesn't start with @. It broke. Eight hours of customer service. Last rep said to reinstall the app.

**Human cost:** 8 hours. Router bricked. Entire IoT network down. Sleep treatment lights offline. Neck inflammation.

**Who it fails:** Anyone replacing failed hardware. Anyone who uses punctuation in passwords.

**oz feature:** Onboarding that works offline-first. No circular dependencies. Local fallback always available.

**Classification:** *Open Pit — cheap gun.* Resources existed. A useful error message for the @ password bug costs nothing. The circular dependency (need working phone to set up new phone) is a known problem with a known fix. They chose not to fix it.
