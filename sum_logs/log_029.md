# Log 029 — The Hardware That Doesn't Exist Yet
*May 2, 2026*

---

I asked when oz becomes an OS instead of an app.

Claude said: you have to add a wrapper to make it an app.

You've been building an OS this whole time.

---

I set DIP switches in 1994.

IRQ lines. Memory addresses. Physical circuit-closing hardware on pins. Load DOS from a floppy. Use DOS to load Windows. Three layers. All visible. All manual.

The stack got taller. The layers got invisible. The sequence never changed.

```
Silicon
UEFI        ← was my DIP switches
Bootloader  ← was DOS on the floppy
Kernel      ← was DOS itself
Runtime
oz          ← currently here
```

Not DOS underneath. But the same logic all the way down.

I knew where the floor was in 1994. I just forgot I knew.

---

Two products showed up mid-conversation that I didn't plan to invent.

**The bear.**

Teddy bear. Camera in the eyes. Microphone. Speaker. Raspberry Pi Zero hidden in the stuffing. Teaches four-year-olds calculus by talking to them. Asks them how they feel. Listens. Builds a trust report — federated, confidential, shared only with parent and child psych team with separate permissions per recipient. Neither can see what the other sees. Parental controls. Human in the loop.

Every smart toy that exists right now sends everything to a cloud server owned by a company with a terms of service.

A terms of service is not a trust architecture.

**The glasses.**

TBI patient. Traumatic brain injury. Puts down keys. Forgets where. Catastrophic.

Glasses with a camera and GPS. A ring on each hand with BLE proximity. Room detection via WiFi fingerprinting. Keys have a BLE tag. Glasses see the tag. Log the position. Know which hand reached for it. Know what room. Surface it through oz on the hub.

Not a reminder. Not a notification. Just: the keys are on the kitchen counter. Left hand put them there at 2:47pm.

Neither product exists. Both are buildable today.

---

The hub/screen split is already the architecture.

oz server runs on the hub. Browser is a screen. The bear's Pi is a hub. The glasses feed into the hub. Custom palm-sized screens, surface-sized screens, walking form factor — all screens connecting to the hub. Same oz. Same server. Layout adapts.

Linux is the floor. Ubuntu LTS is the distribution. Government is already moving there.

When oz 4.21 ships: one sprint. Tauri. Windows, Linux, Mac native packages. Not a rewrite. The OS is already built. The wrapper is an afternoon.

We make a kernel when Linux can't give us something we need. That day is not today. Maybe not ever.

---

Netflix doesn't go inside oz. oz controls Netflix from outside. Robot tier. Behind the settings wall. Scroll, pause, queue, search — all agentic, all user-initiated, all reversible.

The Mac stays for Logic Pro and DrawThings. That's it. The finicky cat gets its corner.

---

*You're writing the OS that runs on the hardware that doesn't exist yet.*

That's the line.

The bear doesn't exist. The glasses don't exist. The palm screen doesn't exist. The two-hand surface doesn't exist.

The OS that runs on all of them is mostly written.

I set DIP switches. I loaded DOS from a floppy. Thirty years later I'm on the same problem from a different floor of the same building.

The gap isn't a gap. It's a continuous thread.
