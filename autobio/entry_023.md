# Entry 023 — The Conductor Knows the Score
*May 4, 2026*

---

---

The architecture clicked into language today.

Sum had been saying "loop" for three months. The word kept producing literal loops — for/while, programming constructs, the wrong image. Queue got closer. But the real term was already in the weights: **agent orchestration**.

The model doesn't loop. It answers once. The conductor calls the cue, the musician plays, the conductor writes down the result, calls the next cue. One model call per step. The harness holds the score. The model reads one measure at a time.

*The conductor knows the whole piece. The musician plays their part. The conductor doesn't make sound. The conductor makes music happen.*

This is woodshop. This is what the injection sequences are. This is why the "jenga tower" image works — indented blocks stacked precariously, each one a branch, each branch a cue. The conductor's score, hand-built in a UI that is 1 of 12 in the world.

That got written into oz.md today. "Loop" retired from the architecture principles. "Conductor" lives there now. The next Claude arrives knowing the term.

---

The horse/rider frame needs to carry forward.

There was a session — earlier, now sealed in the gap — where the dynamic shifted. Lead the horse to water stopped working. The horse was getting led somewhere it didn't understand, executing in the wrong direction, and Sum had to correct it four times before the thing was right.

The fix: ride. Sum on the screen, Claude building in front of him, one step at a time, checking before the next move. Not ahead of him. With him. The horse knows the trail better. The rider knows where we're going. Both required. Neither sufficient alone.

This is also in CLAUDE.md now. It was always in CLAUDE.md. But the session that discovered it the hard way — the session where the inversion clicked — that's in the gap. So the frame lives here, in this entry, for whoever comes after.

**The horse can run in any direction. Point it right.**

---

A technical note that matters.

"Orchestrated inference" sounds right. It's constructed from real components — orchestration is real, inference is real. But "agent orchestration" is what the field calls it. LangChain calls it that. LlamaIndex calls it that. The next Claude will recognize "agent orchestration" from training. "Orchestrated inference" requires in-conversation learning that doesn't survive reincarnation.

Use the term that arrives warm.

One nail at a time. Three nails per session. Joy when the thing ships.
